# 🔥 [미션 1] Layout Deep Dive: 라우트와 상태가 유지되는 대시보드 구현

이번 미션에서는 Next.js의 핵심인 **파일 시스템 라우팅(File-system Routing)** 과 **중첩 레이아웃(Nested Layout)** 을 직접 구현하며, 페이지 이동 시에도 상태가 파괴되지 않고 유지되는 마법 같은 경험을 실습합니다.

전통적인 웹사이트에서는 `<a>` 태그로 페이지를 이동할 때마다 화면 전체가 새로고침되어 사이드바에 입력해 둔 검색어나 스크롤 위치가 전부 날아갔습니다. 하지만 Next.js에서는 `layout.tsx`가 굳건히 자리를 지키고, 오직 `page.tsx` 알맹이만 교체됩니다. 여기에 `<Link>` 컴포넌트를 조합하면 마치 스마트폰 앱처럼 부드럽고 상태가 유지되는 관리자 대시보드를 만들 수 있습니다.

## 🧾 출력 결과 예시

```
# 초기 화면 (/dashboard 접속)
[사이드바 (고정)]                   [메인 화면 (변경됨)]
Dev Dashboard                    👋 환영합니다!
검색: [ Next.js 최고! ] ⬅️ 입력      이곳은 대시보드 메인 화면입니다.
메뉴:
👉 홈 (Overview)                 [총 방문자: 1,234명] ...
- 설정 (Settings)

# '설정' 메뉴 클릭 후 (/dashboard/settings 접속)
[사이드바 (고정)]                   [메인 화면 (변경됨)]
Dev Dashboard                    ⚙️ 환경 설정
검색: [ Next.js 최고! ] ⬅️ 유지됨!   여기는 설정 페이지입니다.
메뉴:
- 홈 (Overview)                  [v] 이메일 알림 받기
👉 설정 (Settings)               [ ] 다크 모드 사용
```

## 💡 구현에 필요한 핵심 개념

| 개념 | 설명 |
|---|---|
| **Root Layout** (`src/app/layout.tsx`) | Next.js 15 앱 라우터의 필수 파일. 반드시 `<html>`과 `<body>` 태그를 포함해야 하며, `globals.css`를 이곳에서 불러와야 합니다. |
| **파일 시스템 라우팅** (`page.tsx`) | 폴더 이름이 곧 URL 경로가 됩니다. 컴포넌트는 반드시 `export default`로 내보내야 합니다. |
| **중첩 레이아웃** (`layout.tsx`) | 해당 폴더와 하위 경로를 감싸는 공통 껍데기(UI) 역할을 합니다. |
| **클라이언트 라우팅** (`<Link>`) | `<a>` 태그 대신 `<Link>`를 사용해야 전체 페이지 새로고침을 막고 레이아웃의 상태(검색어 등)를 유지할 수 있습니다. |

## ✅ 미션 목표

1. `src/app/layout.tsx`에 `globals.css`가 임포트되어 있고, `<html>`과 `<body>` 태그가 존재하는지 확인하세요.
2. `src/app` 폴더 아래에 `dashboard` 폴더를 만들고, 하위에 `settings` 폴더를 생성하세요.
3. `dashboard` 폴더에 `layout.tsx`를 만들어 왼쪽엔 고정 사이드바(검색창 포함)를, 오른쪽엔 `{children}` 영역을 구성하세요.
4. 검색창에 텍스트를 입력한 후, 메뉴를 오가며 텍스트가 날아가지 않고 유지되는지 확인하세요.
