# 🔥 [미션] Layout Deep Dive: 상단 탭 메뉴와 상태가 유지되는 마이페이지 구현

이번 미션에서는 Next.js의 파일 시스템 라우팅과 중첩 레이아웃을 활용하여 쇼핑몰이나 커뮤니티에서 자주 접할 수 있는 **'마이페이지'** 를 만듭니다.

마이페이지에는 보통 상단에 공통 탭 메뉴(프로필, 주문내역, 위시리스트)가 고정되어 있고, 탭을 클릭할 때마다 하단의 내용만 변경됩니다. `<Link>` 컴포넌트를 활용하여 탭을 오갈 때 레이아웃이 다시 렌더링되지 않고, 레이아웃에 포함된 검색창의 텍스트가 파괴되지 않고 그대로 유지되는(상태 보존) 경험을 실습해 봅니다.

---

## 🧾 출력 결과 예시

```
# 초기 화면 (/mypage 접속)
[상단 탭 메뉴 (고정)]
내 검색: [ 배송조회 ] ⬅️ 입력
[ 👉 프로필 ] [ 주문 내역 ] [ 위시리스트 ]
-----------------------------------------
[메인 화면 (변경됨)]
👤 내 프로필 정보
이름: 김개발
등급: VIP 회원

# '주문 내역' 탭 클릭 후 (/mypage/orders 접속)
[상단 탭 메뉴 (고정)]
내 검색: [ 배송조회 ] ⬅️ 탭을 바꿔도 입력한 글자가 유지됨!
[ 프로필 ] [ 👉 주문 내역 ] [ 위시리스트 ]
-----------------------------------------
[메인 화면 (변경됨)]
📦 내 주문 내역
- 기계식 키보드 (배송완료)
- 무선 마우스 (배송중)
```

---

## 💡 구현에 필요한 핵심 개념

| 개념 | 설명 |
|---|---|
| **Root Layout** (`src/app/layout.tsx`) | Next.js 15 앱 라우터의 필수 파일. 반드시 `<html>`과 `<body>` 태그를 포함해야 하며, `globals.css`를 이곳에서 불러와야 합니다. |
| **파일 시스템 라우팅** (`page.tsx`) | 폴더의 깊이가 곧 URL 경로의 깊이가 됩니다. (예: `/mypage/orders`) |
| **중첩 레이아웃** (`layout.tsx`) | 탭 메뉴와 같이 여러 하위 페이지에서 공통으로 사용되는 UI를 정의하고 `{children}`을 통해 개별 페이지의 내용을 렌더링합니다. |
| **클라이언트 라우팅** (`<Link>`) | 브라우저 새로고침 없이 필요한 컴포넌트만 부드럽게 교체하여 사용자 경험(UX)을 향상시키고 레이아웃의 상태를 유지합니다. |
