## 1. 스타일링 전략 Overview

React에서 스타일을 적용하는 대표적인 방법 **5가지** 한눈에 보기:

```jsx
// 1) Vanilla CSS -- 전역 적용
import "./App.css";
<h1>안녕하세요</h1>

// 2) CSS Modules -- 파일별 스코프 격리
import styles from "./Button.module.css";
<button className={styles.button}>클릭</button>

// 3) styled-components -- JS 안에서 스타일 선언
const Title = styled.h1`color: purple;`;
<Title>안녕하세요</Title>

// 4) Inline Style -- JSX style 속성에 객체 전달
<h1 style={{ color: "red", fontSize: "20px" }}>인라인</h1>

// 5) Tailwind CSS -- 유틸리티 클래스 조합
<button className="bg-blue-500 text-white p-2 rounded">저장</button>
```

## 2. Vanilla CSS -- 전역 스타일의 장단점

CSS 파일을 작성하고 컴포넌트에서 `import` 한다.

```css
/* App.css */
h1 { color: blue; text-align: center; }
button { background-color: green; color: white; padding: 10px; border-radius: 5px; }
```

```jsx
import "./App.css";

export default function App() {
  return (
    <div>
      <h1>안녕하세요, React!</h1>
      <button>클릭</button>
    </div>
  );
}
```

**핵심 특징**

- `import "./styles.css"` 하면 **전역으로 적용** -- 모든 컴포넌트에 영향
- 같은 클래스명이 여러 CSS 파일에 있으면 **나중에 import된 파일이 덮어씀**
- 간단하지만 프로젝트가 커지면 **스타일 충돌 위험**이 높아짐

> 장점: 익숙하고 단순 / 단점: 클래스명 충돌, 전역 오염

---

## 3. CSS Modules -- 스코프 격리 (.module.css)

파일명을 `*.module.css`로 작성하면 빌드 시 클래스명이 **고유 해시값으로 변환**된다.

```
.button  -->  button___3Gh7x  (빌드 후)
```

```css
/* Button.module.css */
.button { background-color: green; color: white; padding: 10px; border-radius: 5px; }
```

```jsx
import styles from "./Button.module.css";

export default function Button({ children }) {
  return <button className={styles.button}>{children}</button>;
}
```

**전역 CSS와의 차이**

```jsx
// 전역 CSS -- 충돌 발생
import "./Header.css";   // .button { color: red; }
import "./Footer.css";   // .button { color: blue; }  <-- 나중 import가 이김

// CSS Modules -- 충돌 없음
<Button>클릭(모듈)</Button>               // button___3Gh7x 적용
<button className="button">전역 버튼</button> // 전역 .button 적용
```

> 장점: 클래스 충돌 방지, 컴포넌트 캡슐화 / 단점: 동적 스타일링이 불편

## 4. styled-components -- CSS-in-JS

### 4-1. 기본 + props 동적 스타일링

```bash
npm install styled-components
```

```jsx
import styled from "styled-components";

const Button = styled.button`
  background: ${({ variant }) =>
    variant === "primary" ? "royalblue" :
    variant === "danger"  ? "crimson"   : "gray"};
  color: white;
  border: none;
  border-radius: 8px;
  padding: ${({ size }) => (size === "lg" ? "14px 18px" : "10px 14px")};
  font-size: ${({ size }) => (size === "lg" ? "16px" : "14px")};
  cursor: pointer;
`;

// 사용
<Button variant="primary" size="lg">저장</Button>
<Button variant="danger">삭제</Button>
<Button>기본</Button>
```

- `styled.태그명` + 템플릿 리터럴로 스타일 정의
- **props**를 CSS 내부에서 직접 참조 -- 동적 스타일링의 핵심
- 런타임에 고유 클래스명 자동 생성 -- 전역 오염 없음

### 4-2. state 연동 -- 토글 버튼

```jsx
import { useState } from "react";
import styled from "styled-components";

const Toggle = styled.button`
  background: ${({ active }) => (active ? "seagreen" : "gray")};
  color: white;
  padding: 10px 12px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
`;

export default function ToggleDemo() {
  const [on, setOn] = useState(false);
  return (
    <Toggle active={on} onClick={() => setOn((v) => !v)}>
      {on ? "ON" : "OFF"}
    </Toggle>
  );
}
```

### 4-3. Pseudo-class & 미디어쿼리

```jsx
const Action = styled.button`
  background: steelblue;
  color: white;
  cursor: pointer;

  &:hover  { background: dodgerblue; }
  &:active { background: navy; transform: scale(0.97); }

  @media (max-width: 480px) {
    padding: 8px;
    font-size: 12px;
  }
`;
```

### 4-4. ThemeProvider -- 다크모드

```jsx
import styled, { ThemeProvider } from "styled-components";

const light = { bg: "#ffffff", fg: "#111111" };
const dark  = { bg: "#111111", fg: "#ffffff" };

const Page = styled.div`
  background: ${({ theme }) => theme.bg};
  color: ${({ theme }) => theme.fg};
  min-height: 100vh;
  padding: 24px;
`;

export default function ThemedApp({ darkMode }) {
  return (
    <ThemeProvider theme={darkMode ? dark : light}>
      <Page>테마에 따라 배경과 글자색이 달라집니다.</Page>
    </ThemeProvider>
  );
}
```

- `ThemeProvider`로 감싸면 모든 하위 styled 컴포넌트에서 `theme` 접근 가능
- props로 `darkMode`를 전달하면 전역 테마를 한 번에 교체

> 장점: 완전한 캡슐화, props 동적 제어, 테마 시스템 /
> 단점: 런타임 CSS 생성 비용, JS 번들 크기 증가

## 5. 인라인 스타일 -- camelCase, 한계점

```jsx
export default function Welcome() {
  const titleStyle = {
    color: "royalblue",
    fontSize: "24px",       // camelCase 필수
    fontWeight: "bold",
    padding: "10px"
  };
  return <h1 style={titleStyle}>안녕하세요, React!</h1>;
}
```

**camelCase 변환 규칙**

| CSS 속성 | 인라인 스타일 |
|---------|-------------|
| `background-color` | `backgroundColor` |
| `font-size` | `fontSize` |
| `border-radius` | `borderRadius` |

- `style` 속성에는 **문자열이 아닌 객체**를 전달
- 단위는 **문자열**로 작성: `"24px"`, `"50%"`
- 이중 중괄호 `{{ }}` -- 바깥은 JSX 표현식, 안쪽은 객체 리터럴

**한계점**

| 가능 | 불가능 |
|------|--------|
| color, padding 등 기본 속성 | `:hover`, `:focus` pseudo-class |
| props/state 기반 동적 값 | `@media` 미디어쿼리 |
| 빠른 프로토타이핑 | `@keyframes` 애니메이션 |

> 인라인 스타일은 **빠른 실험 / 소규모 동적 스타일**에만 적합하다.

## 6. UI 라이브러리 -- Tailwind vs MUI vs Ant Design 비교

| 구분 | Tailwind CSS | Material UI (MUI) | Ant Design |
|------|-------------|-------------------|------------|
| **유형** | 유틸리티 CSS 프레임워크 | 컴포넌트 라이브러리 | 컴포넌트 라이브러리 |
| **설계 철학** | 클래스 조합으로 빠른 조립 | Google Material Design | 엔터프라이즈 UI |
| **스타일 작성** | className에 유틸리티 클래스 | props로 variant 제어 | props로 type 제어 |
| **커스터마이징** | tailwind.config.js | ThemeProvider | ConfigProvider |
| **적합한 프로젝트** | 커스텀 디자인 | 표준화된 SPA | 관리자 대시보드 |

### Tailwind CSS

```jsx
<button className="bg-blue-500 text-white p-2 rounded hover:bg-blue-600">클릭</button>
```

| 클래스 | 의미 |
|--------|------|
| `bg-blue-500` | 배경색 blue |
| `text-white` | 글자색 흰색 |
| `p-2` | padding 0.5rem |
| `rounded` | border-radius |
| `hover:bg-blue-600` | hover 시 배경 변경 |

### Material UI (MUI)

```jsx
import Button from "@mui/material/Button";

<Button variant="contained" color="primary">클릭</Button>
```

### Ant Design

```jsx
import { Button } from "antd";

<Button type="primary">클릭</Button>
```

> Tailwind = 빠른 조립, MUI = 세련된 디자인 표준, Ant Design = 비즈니스형 UI

## 7. react-icons -- 간편한 아이콘 사용

```bash
npm install react-icons
```

```jsx
import { useState } from "react";
import { FaHeart } from "react-icons/fa";

export default function LikeButton() {
  const [liked, setLiked] = useState(false);
  return (
    <button
      onClick={() => setLiked((prev) => !prev)}
      style={{
        background: "none", border: "none", cursor: "pointer",
        fontSize: "20px", color: liked ? "crimson" : "gray",
      }}
    >
      <FaHeart /> {liked ? "좋아요 취소" : "좋아요"}
    </button>
  );
}
```

**주요 아이콘 팩**

| 접두사 | 라이브러리 | 예시 |
|--------|-----------|------|
| `Fa` | Font Awesome | `FaHeart`, `FaUser` |
| `Md` | Material Design | `MdHome`, `MdSettings` |
| `Ai` | Ant Design Icons | `AiOutlineSearch` |
| `Bi` | BoxIcons | `BiMenu`, `BiShare` |
| `Hi` | Heroicons | `HiOutlineX` |

## 면접 핵심 Q&A

### Q. CSS Modules와 일반 CSS의 차이?

> **A.** 일반 CSS는 `import`하면 전역으로 적용되어 같은 클래스명이 충돌할 수 있다.
> CSS Modules는 `*.module.css`로 지정하면 빌드 시 클래스명에 **고유 해시가 추가**되어
> (`button` -> `button___3Gh7x`) 파일 단위로 스코프가 격리된다.

### Q. CSS-in-JS의 장단점?

> **A.**
> - **장점**: props/state 기반 동적 스타일링이 자연스럽고, 고유 클래스명으로 완전한 캡슐화가 가능하다. ThemeProvider로 전역 테마 관리가 편리하다.
> - **단점**: 런타임 CSS 생성으로 **성능 오버헤드**가 있고, JS 번들 크기가 증가한다.

### Q. 인라인 스타일의 한계?

> **A.** `:hover`, `:focus` pseudo-class 사용 불가, `@media` 미디어쿼리와 `@keyframes` 애니메이션도 적용할 수 없다. 스타일 재사용이 어렵고 유지보수성이 떨어진다.

### Q. Tailwind CSS의 특징?

> **A.** 유틸리티 클래스를 조합하여 스타일을 적용하는 방식이다.
> PurgeCSS로 미사용 클래스를 제거하여 번들을 최소화하고,
> `hover:`, `md:` 접두사로 상태/반응형 스타일을 지원한다.
> 단, 클래스가 길어지면 가독성이 떨어질 수 있다.

### Q. 프로젝트에서 스타일링 전략을 선택하는 기준?

> **A.**
> - **소규모 / 빠른 프로토타이핑** -> Inline Style 또는 Vanilla CSS
> - **중규모 / 클래스 충돌 방지** -> CSS Modules
> - **동적 스타일이 많은 SPA** -> styled-components (CSS-in-JS)
> - **빠른 개발 + 커스텀 디자인** -> Tailwind CSS
> - **표준 디자인 시스템 필요** -> MUI 또는 Ant Design
>
> 핵심 판단 기준: **프로젝트 규모**, **동적 스타일 빈도**, **팀 기술 스택**, **성능 요구사항**

## 용어 글로서리

| 용어 | 설명 |
|------|------|
| **스코프 (Scope)** | 스타일이 영향을 미치는 범위. 전역(Global) vs 지역(Local) |
| **CSS Modules** | `.module.css` 확장자로 빌드 시 고유 클래스명을 생성하는 방식 |
| **CSS-in-JS** | JS 코드 안에서 CSS를 작성하는 패턴 (styled-components, Emotion) |
| **ThemeProvider** | styled-components의 Context 기반 테마 전달 컴포넌트 |
| **유틸리티 CSS** | 하나의 클래스가 하나의 CSS 속성을 담당하는 설계 방식 |
| **PurgeCSS** | 빌드 시 미사용 CSS 클래스를 제거하여 번들 최적화 |
| **camelCase** | JS에서 CSS 속성명 표기 방식 (`fontSize`, `backgroundColor`) |
| **Pseudo-class** | `:hover`, `:focus`, `:active` 등 요소 상태 선택자 |
| **Tagged Template Literal** | `` styled.div`...` `` 형태의 JS 문법 |

## 자가 점검 체크리스트

- [ ] Vanilla CSS의 전역 적용 특성과 충돌 문제를 설명할 수 있다
- [ ] CSS Modules의 `.module.css` 네이밍과 해시 클래스명 원리를 이해한다
- [ ] `styles.button` 형태로 CSS Modules 클래스를 적용할 수 있다
- [ ] styled-components로 props 기반 동적 스타일을 작성할 수 있다
- [ ] `&:hover`, `@media`를 styled-components 안에서 사용할 수 있다
- [ ] ThemeProvider로 다크/라이트 테마 전환 구조를 구현할 수 있다
- [ ] 인라인 스타일의 camelCase 규칙과 한계점을 알고 있다
- [ ] Tailwind CSS의 유틸리티 클래스 조합 방식을 이해한다
- [ ] MUI, Ant Design의 특징과 적합한 사용 사례를 구분할 수 있다
- [ ] react-icons를 설치하고 아이콘 컴포넌트를 사용할 수 있다
- [ ] 프로젝트 특성에 따라 적절한 스타일링 전략을 선택하는 기준을 설명할 수 있다
