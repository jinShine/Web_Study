# 🔥 [미션] 통합 비즈니스 솔루션 "Nexus"

## 🏢 프로젝트 배경

- **서비스명:** Nexus (전사 자원 관리 대시보드)
- **핵심 과제:**
  - 앱 전체에 공통으로 적용되는 전역 알림 바(Root Layout)가 있어야 함.
  - 대시보드 영역에 진입하면 사이드바(Dashboard Layout)가 나타나야 함.
  - 대시보드 내 설정 페이지 진입 시, 설정 전용 상단 탭(Settings Layout)이 추가로 나타나야 함.

---

## 🎯 기술적 해결 목표

- **중첩 레이아웃(Nested Layouts):** `Root` → `Dashboard` → `Settings`로 이어지는 **3단계 레이아웃 중첩 구조** 구현.
- **독립적 상태 유지:** 대시보드 사이드바의 입력값과 설정 페이지 상단 탭의 선택 상태가 페이지 이동 중에도 각각 유지되는지 검증.
- **레이아웃 격리:** 대시보드 외부(예: 메인 랜딩 페이지)에서는 대시보드용 사이드바가 절대 보이지 않도록 격리.

---

## 🛠️ 상세 구현 사항 (Implementation Requirements)

### [Phase 1] 전역 베이스 구축 (`src/app/layout.tsx`)

- **전역 공지 바:** 모든 페이지(로그인, 메인, 대시보드 포함) 상단에 고정된 Announce Bar를 만듭니다.
- **상태 확인용 입력창:** 여기에 글자를 입력하고 대시보드 내부로 이동해보고 대시보드 밖으로 나갔을 때도 글자가 유지되는지 확인합니다.

### [Phase 2] 메인 랜딩 페이지 구축 (`src/app/page.tsx`)

- **레이아웃 격리 확인:** 이 페이지에서는 대시보드의 사이드바가 보이지 않아야 하며, 오직 전역 공지 바만 유지되어야 합니다.

### [Phase 3] 대시보드 전용 레이아웃 (`src/app/dashboard/layout.tsx`)

- **L-사이드바 구조:** 대시보드 하위의 모든 페이지(`/dashboard`, `/dashboard/settings` 등)에만 적용되는 사이드바를 구축합니다.
- **불변성 테스트:** 강의 스크립트와 동일하게 검색창을 배치하여 상태 유지 여부를 확인합니다.

### [Phase 4] 설정 전용 중첩 레이아웃 (`src/app/dashboard/settings/layout.tsx`)

- **상단 탭 네비게이션:** 설정 메뉴 내에서만 보이는 상단 탭(프로필 설정, 보안 설정)을 만듭니다.
- **계층 구조의 정수:** 이 레이아웃은 Dashboard Layout의 `children` 안에서 렌더링되어야 합니다.
